HTML

  • The head element contains information about the webpage. The contents inside are usually not visible (aside from the title)
  • The body element represents the visible content shown to the user.
  • Focus on using the proper elements for the job so that the page is more accessible for screen readers and other assisted browsing technologies.
  • Use the class and id attributes to provide pointers for CSS and JS to reference

CSS

Positioning

    position: static;

  • Default Value. Properties have no effect, and element is positioned according to the normal flow of the document.

    • position: relative;

  • The element is positioned according to the normal flow of the document, and then offset relative to itself based on the values of top, right, bottom, and left properties. The offset does not affect the position of any other elements, so the space given for the element prior to being moved is as if the position were static.

    • position: absolute;

  • The element is removed from the normal document flow, and no space is created for the element in the page layout. The element is positioned relative to its closest positioned ancestor (if any) or to the initial containing block. Its final position is determined by the values of top, right, bottom, and left.

    • position: fixed;

  • The element is removed from the normal document flow, and no space is created for the element in the page layout. The element is positioned relative to its initial containing block, which is the viewport in the case of visual media. Its final position is determined by the values of top, right, bottom, and left.
  • Multiple elements can be targeted at once. Useful for making changes to certain elements that you want to have sharing styles.
  • * is used to select EVERY element. Use sparingly
  • Using a seperate file for your style sheet is best practice when working on large projects, as it keeps all the CSS in one place and means you will have to do less work to make changes acrossed the entire project.

Git (Go on.... git)

  • NEVER work off of the main branch. Always create a seperate branch.
        • Workflow goes like this: (Type these into terminal)
    1. Git Pull origin main (Creates an up to date local copy of the main branch)
    2. Git checkout -b nameOfNewBranch (Creates and switches you to new branch)
    3. Do your work on this new branch
    4. git add . (the . adds all of the current directory to the staging area to be commited)
    5. Git commit -m message for what you did(Save your work, and do this often)
    6. Git push (Pushes your saved copy to the branch on github)
    7. On github, create a pull request and then merge
